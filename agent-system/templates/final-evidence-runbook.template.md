# Final Evidence Runbook

Use this runbook before claiming external, live, independent, production, public, or client-ready proof.

## Rules

- Do not read or print secret values.
- Do not run network, paid provider, external write, destructive, or subagent work without approval for scope and budget.
- Record approval before the run.
- Preserve raw evidence only in approved evidence folders.
- Keep bounded structured observations.
- Run final audit after evidence is added.
- A descope is a limitation, not proof.

## Approval Record

Required fields:

- action id;
- provider/tool/reviewer scope;
- item count or max requests;
- maximum cost or time budget;
- approved by;
- approval ref;
- approved at;
- expiry or review date.

## Evidence Linkage

Every pass condition should identify:

- machine artifact paths;
- raw refs when raw evidence exists;
- row counts or item counts;
- checker/report refs;
- claim boundary impact.

Markdown reports alone are not enough for `external_pass`.
