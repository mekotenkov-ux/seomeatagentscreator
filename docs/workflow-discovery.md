# Workflow discovery перед созданием агента

`grill-me` отвечает на вопрос: какой агент нужен, для кого, с какими рисками и границами.

`workflow-loop-me` отвечает на другой вопрос: какие повторяемые рабочие процессы реально нужно делегировать, как они запускаются, какие входы используют, где нужен человек и чем доказывается завершение.

## Зачем это нужно

Люди часто просят "сделай агента", но описывают не процесс, а желаемую магию. Workflow discovery заставляет сначала найти повторяемые loops: то, что происходит каждую неделю, каждый день, по событию, по входящему письму, issue, отчету, клиентскому запросу, релизу или проверке.

Агент проектируется только после того, как выбран хотя бы один workflow, который можно реализовать без догадок.

## Минимальный порядок

1. Запустите `grill-me` и зафиксируйте цель, пользователей, scope, автономию, риски, источники, платформы, экспорт, валидацию и done condition.
2. Запустите `workflow-loop-me`.
3. Соберите `workflow-notes.md`: язык пользователя, каналы, инструменты, повторяемые задачи, боли.
4. Составьте ledger candidate loops.
5. Выберите первый workflow: частый, ценный, проверяемый, с понятным trigger и done signal.
6. Заполните `workflow-spec.template.md`.
7. Только потом переходите к IR, router, skills, tools и evals.

## Что должен содержать workflow spec

- Trigger: событие или расписание.
- Inputs: что приходит на вход и как валидируется.
- Actors: люди, агентские роли, внешние системы.
- Tools/channels: где процесс живет.
- State: что нужно помнить между runs.
- Artifacts: что создается или меняется.
- Steps: фактическая последовательность работы.
- Checkpoints: где нужен человек.
- Brief: что человек должен увидеть для быстрого решения.
- Failure modes: как процесс ломается.
- Validation: done signal, checks, not-done report.
- Budgets: шаги, tools, runtime, cost, retry.

## Правило готовности

Workflow spec готов только когда fresh implementer agent может построить его без уточняющих вопросов. Если остается вопрос, это не готовый spec, а draft.
