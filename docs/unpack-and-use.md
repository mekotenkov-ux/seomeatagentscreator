# Как распаковывать и использовать

## Вариант 1: как основу нового agent-dev репозитория

1. Создайте новый репозиторий.
2. Скопируйте туда `agent-system/`.
3. Добавьте свой `package/` и `devkit/`.
4. Используйте `agent-system/AGENTS.md` как инструкцию для агента-создателя.

## Вариант 2: как reference в существующем проекте

1. Скопируйте `agent-system/` в проект.
2. Не смешивайте его с доменными агентами.
3. Доменные правила кладите в отдельные scoped references.
4. Универсальное ядро обновляйте только устойчивыми уроками, которые применимы к разным агентам.

## Вариант 3: как публичный шаблон

1. Сделайте fork.
2. Переименуйте package metadata.
3. Оставьте MIT или замените лицензию осознанно.
4. Удалите все доменные примеры перед публикацией.

## Что делать после распаковки

- заполнить IR;
- собрать router;
- создать focused skills;
- описать tools;
- добавить checkers;
- создать trigger/output evals;
- провести release review;
- собрать clean runtime;
- отдельно собрать devkit;
- проверить архив или public repo.
