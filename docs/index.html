Что внутри

Практический конструктор. Шаблоны, правила, чеклисты и runtime-инструкции для разработки надежных agent workflows.

Подходит для Codex, Claude Code, Cursor, локальных CLI-пакетов и командных процессов, где нужны проверяемые действия, артефакты и границы ответственности.

Создано в школе "SEO Мясо"

Подход вырос из прикладной работы и образовательной практики SEOMeat: системного обучения, проектирования AI-процессов, проверки качества агентских действий и упаковки воспроизводимых workflows. Подробнее о школе: seomeat.ru.

Что добавлено в ядро

Skill Training Lab

Skill candidates, fresh-context forward testing, promotion gates и living adaptation вместо скиллов, написанных под текущий чат.

target conformance

Adapters для Codex, Claude, Cursor, CLI или других runtimes проверяются против одного Agent/Skill IR.

final evidence

Local proof, external proof, descopes, independent reviews и forbidden claims разделены машинным контрактом.

Runtime/devkit

Clean staging, FILES.sha256, zip entry inspection, install simulation и запрет на dev/test/raw artifacts в runtime.

birth protocol

Первый запуск разделен на runtime detection, active adapter decision, environment readiness, project context и birth gate.

workflow discovery

Перед IR агент вытаскивает реальные loops: trigger, inputs, actors, tools, checkpoints, artifacts, failure modes и done signal.

Процесс

1

Preflight

Фиксируем цель, пользователя, scope, автономию, риски, источники, платформы, экспорт и done condition.

2

IR до адаптеров

Сначала создаем платформенно-нейтральный Agent/Skill IR, потом Codex, Claude, Cursor или CLI-адаптеры.

3

Проверяемые слои

Добавляем tool registry, permission matrix, trigger lab, output eval lab, checkers и release review.

Отладка

Для качественной отладки создавайте отдельный проект под конкретного агента, подключайте туда runtime и работайте из отдельного чата. Агент-создатель должен видеть полный лог, state files, checker reports и generated artifacts, чтобы чинить систему по реальному trace.
